# How to Contribute

Fudis Design System is open for new development contributions in - and outside of Funidata company. 

## Before contributing

**All Fudis Design System proposals must be generic and they must solve commonly repeated challenges across learning services.** If the proposal is considered as project spesific, it will be discarded.
Therefore, it's important to define the scope of your proposal â€” whether it's an improvement to an existing component or a completely new feature. The best way to get started is by becoming familiar with the existing Fudis component library.
Contributions can also include updates to existing documentation and bug fixes. Do not hesitate to contact Fudis Team in advance for help and guidance.

## Contribution guidelines (external developer):
- Define your implementation proposal.
- Contact Fudis team in order to find best way contribute. 
  - **Note!** New features require a design that Fudis team will provide consultancy on.
- Get familiar with the Fudis development practices as documented in [DS Practices](/docs/documentation-development-ds-practises-ways-of-working--documentation).
- [Fork the repository](). Select yourself as the owner and press Create fork.
- Set up your local development environment as documented in [Getting Started](/docs/documentation-development-getting-started--documentation)
- Create a new branch, as guided in [Git and Github Convension](/docs/documentation-development-tooling-git-and-github-conventions--documentation)
- Create your implementation including the following:
  - Code and Design
  - Unit tests and Visual regression tests
  - Documentation
- Make a pull request on [Fudis Github page](https://github.com/funidata/fudis).
- Fudis team will review the PR, merge it or request for new changes. All open PRs are reviewed to satisfy the following acceptance criteria: Accessibility, Usability, Security, Modularity and Performance.

## Contribution guidelines (Funidata developer):
- Propose your implementation by contacting Fudis team at Funidata's [#fudis Slack channel](https://sisudev.slack.com/archives/C03G46K6LG1).
