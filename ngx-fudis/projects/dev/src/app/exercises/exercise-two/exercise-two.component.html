Description Fudis Design System is an open source project and we welcome contributions from everyone. This exercise guides you through the process of contributing to Fudis Design System. The whole process is documented on page . On this exercise we will be closely looking at steps 4/5 "Implementation step-by-step" onwards. All the necessary material is provided in a downloaded package in a link below. For this exercise you are not required to write component unit tests or visual regression tests that are otherwise mandatory when exposing a new library component. Outcome With the provided material, you will be implementing a Fudis Dialog to Dummy component. This Dummy component is exposed as a Storybook story. As a dialog content you will be implementing a Dummy Table component that is styled by utilising Core Style Library. After completing the exercise we encourage you to create a pull request that our Fudis team will provide feedback on. Instructions
  1. Download the provided content, that includes a Dummy folder.
  2. Move the downloaded Dummy folder under ngx-fudis project ngx-fudis/src/lib/components.
  3. Start Storybook by running npm start in root directory. Navigate to . When Storybook is running, you should have Dummy folder showing on components sidepanel listing. This folder contains Example story (dummy.stories.ts) and documentation page (dummy.mdx). The Example story is currently empty.
  4. To get started with your implementation add Dummy component selector ( step 1 ) to the story template in dummy.stories.ts. You should see the dummy component showing up in Storybook.
  5. When you have your story showing up, you are free continue with the implementation in order you like. However, we recommend that you complete the steps in each file in order to produce a fully functional story example with documentation.
  6. As a bonus task, create a styling for Dummy Table component that matches dummy-table-reference.png by utilising Core Style Library. You can find the documentation for the library here: .
  7. When you have completed the exercise, make a pull request on Fudis Github repository. Name your pull request as "Exercise 2" and add a label "Exercise", so we can easily recognise it as practice material.
